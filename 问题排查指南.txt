===============================================
   股票气泡 - 问题排查指南
===============================================

📋 显示"Err"时的排查步骤
========================

第一步：查看错误提示
-------------------
1. 鼠标悬停在气泡上，会显示详细错误信息
2. 注意查看边框颜色：
   🟠 橙色 = 数据获取失败
   🔴 红色 = 网络或系统错误


第二步：检查股票代码格式
-----------------------
正确格式示例：
  ✅ sz302132  （深圳股票）
  ✅ sh600745  （上海股票）

错误格式示例：
  ❌ 302132    （缺少 sz/sh 前缀）
  ❌ SZ302132  （大小写错误，应为小写）
  ❌ sz 302132 （有空格）
  ❌ 0302132   （前缀错误）


第三步：检查网络连接
-------------------
1. 确保电脑已连接互联网
2. 打开浏览器访问：https://qt.gtimg.cn
3. 检查防火墙是否拦截 StockBubble.exe
4. 尝试右键点击气泡 → 刷新


第四步：查看详细日志
-------------------
1. 右键点击气泡
2. 选择"查看错误日志"
3. 记录错误信息


第五步：验证股票代码
-------------------
1. 访问证券交易所网站确认代码
2. 检查股票是否停牌
3. 确认是否交易时间：
   • 交易日：周一至周五
   • 交易时间：9:30-11:30, 13:00-15:00
   • 节假日休市


常见错误类型
============

1. "股票代码格式错误"
   → 检查格式：必须是 sz/sh + 6位数字

2. "网络请求失败"
   → 检查网络连接
   → 检查防火墙设置

3. "请求超时"
   → 网络速度慢或不稳定
   → 稍后重试

4. "数据格式不正确"
   → 股票代码可能不存在
   → 验证代码是否正确

5. "价格数据无效"
   → 可能是市场未开盘
   → 可能是股票停牌

6. "股票代码可能错误或已停牌"
   → 价格为0，检查代码


测试建议
========

使用以下活跃股票测试：
  • sz000001  （平安银行）
  • sh600036  （招商银行）
  • sz300059  （东方财富）
  • sh688981  （中芯国际）


日志文件位置
============
%AppData%\StockBubble\error.log

可以手动打开查看完整错误记录


仍然无法解决？
==============
1. 保存错误日志
2. 检查 %AppData%\StockBubble\config.txt
3. 删除配置文件重新设置
4. 重启程序


提示
====
• 首次运行建议使用常见股票代码测试
• 确保在交易时间测试
• 保持网络连接稳定
• 如长时间显示"..."表示正在请求数据
